O projekcie

Resumo to pełnoprawna aplikacja webowa zbudowana w Django — nowoczesny kreator CV w stylu SaaS. Użytkownik rejestruje konto (z weryfikacją e-mail), wypełnia kreator CV krok po kroku, wybiera jeden z trzech szablonów wizualnych i pobiera gotowy dokument jako plik PDF.

Projekt wykracza poza statyczny HTML — używa backendu Django, bazy danych PostgreSQL hostowanej na Neon, wysyłki e-maili przez SMTP oraz generowania PDF przez bibliotekę WeasyPrint.

Z perspektywy wymagań projekt zawiera ponad 10 różnych typów formularzy HTML rozmieszczonych w różnych częściach aplikacji — od rejestracji i logowania, przez kreator CV, aż po ustawienia konta.

Wymagania — 10 typów formularzy HTML

Jedynym formalnym wymaganiem było zastosowanie minimum 10 różnych typów pól formularzy HTML. Poniżej lista wszystkich zastosowanych typów wraz z miejscem ich użycia w projekcie.

type="text" — imię, nazwisko, tytuł zawodowy, nazwa firmy, stanowisko, miasto, kraj, szkoła, kierunek
type="email" — rejestracja, logowanie, dane kontaktowe w CV
type="password" — rejestracja (hasło + potwierdzenie), logowanie, zmiana hasła, reset hasła
<textarea> — podsumowanie zawodowe w CV, opis obowiązków, opis projektu
<select> — wybór szablonu CV (Klasyczny / Minimalistyczny / Nowoczesny), poziom języka, poziom umiejętności
type="checkbox" — „Zapamiętaj mnie" przy logowaniu, „Praca do teraz" (brak daty końcowej), zgoda na regulamin
type="radio" — wybór widoczności CV (publiczne / prywatne), wybór koloru akcentu szablonu
type="date" — daty zatrudnienia (od / do), daty edukacji, daty projektów
type="tel" — numer telefonu w danych kontaktowych CV
type="url" — link do LinkedIn, GitHub, portfolio, strony projektu
type="file" — upload zdjęcia profilowego do CV
type="range" — suwak poziomu umiejętności (0–100%) w sekcji Skills
type="search" — wyszukiwanie szablonów i CV na dashboardzie

2 Weryfikacja e-mail — token w URL 

# views.py — generowanie tokenu i wysyłka e-maila
import secrets
from django.core.mail import send_mail

def register(request):
    if request.method == 'POST':
        token = secrets.token_urlsafe(32)
        user.verification_token = token
        user.save()

        verify_url = request.build_absolute_uri(
            f'/users/verify/{token}/'
        )
        send_mail(
            subject='Potwierdź konto w Resumo',
            message=f'Kliknij: {verify_url}',
            from_email='noreply@resumo.pl',
            recipient_list=[user.email]
        )

3 Generowanie PDF — WeasyPrint + Django 

from weasyprint import HTML
from django.template.loader import render_to_string
from django.http import HttpResponse

def download_pdf(request, cv_id):
    cv = CV.objects.get(id=cv_id, user=request.user)

    # Renderuj szablon HTML do stringa
    html_string = render_to_string(
        f'cv/templates/{cv.template}.html',
        {'cv': cv}
    )

    # Konwertuj HTML → PDF przez WeasyPrint
    pdf_file = HTML(string=html_string).write_pdf()

    # Zwróć jako plik do pobrania
    response = HttpResponse(
        pdf_file,
        content_type='application/pdf'
    )
    response['Content-Disposition'] = \
        f'attachment; filename="cv_{cv.id}.pdf"'
    return response

4 Automatyczne usuwanie kont — APScheduler 

# management/commands/delete_unverified_users.py
# Niezweryfikowane konta usuwane automatycznie po 15 min

from django.utils import timezone
from datetime import timedelta

def delete_unverified():
    cutoff = timezone.now() - timedelta(minutes=15)
    deleted = CustomUser.objects.filter(
        is_verified=False,
        date_joined__lt=cutoff
    ).delete()
    print(f'Usunięto {deleted[0]} niezweryfikowanych kont')

# APScheduler uruchamia tę funkcję co 5 minut
scheduler.add_job(
    delete_unverified,
    'interval',
    minutes=5
)
