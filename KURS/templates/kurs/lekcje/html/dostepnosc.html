Dostępność (Accessibility)

Dostępność stron internetowych (web accessibility, w skrócie a11y) to projektowanie stron tak, żeby mogły z nich korzystać wszyscy użytkownicy — w tym osoby z niepełnosprawnościami wzroku, słuchu, ruchu czy poznawczymi. To nie tylko kwestia etyki — w wielu krajach jest wymogiem prawnym.

ARIA — Accessible Rich Internet Applications

Atrybuty ARIA dodają semantykę elementom, które jej nie mają. Używaj ich gdy HTML5 nie oferuje odpowiedniego znacznika: 

<!-- Rola elementu -->
<div role="button" tabindex="0">Kliknij mnie</div>

<!-- Etykieta dla elementu bez widocznego tekstu -->
<button aria-label="Zamknij okno">×</button>

<!-- Opis przez inny element -->
<input type="text" aria-describedby="wskazowka">
<span id="wskazowka">Wpisz co najmniej 8 znaków</span>

<!-- Stan elementu -->
<button aria-expanded="false">Menu</button>
<div aria-hidden="true">Dekoracja, ignoruj</div>

Zasady dobrej dostępności HTML

1. Używaj semantycznych znaczników

<!-- ❌ Źle -->
<div onclick="...">Kliknij mnie</div>

<!-- ✅ Dobrze -->
<button type="button">Kliknij mnie</button>

2. Zawsze dodawaj alt do obrazów

<img src="wykres.png" alt="Wykres słupkowy: sprzedaż wzrosła o 30% w Q3 2026">
<img src="dekoracja.png" alt=""> <!-- Dekoracyjny — pusty alt -->

3. Etykiety formularzy

<!-- ❌ Źle -->
<input type="text" placeholder="Imię">

<!-- ✅ Dobrze -->
<label for="imie">Imię:</label>
<input type="text" id="imie" name="imie">

4. Kontrast kolorów

Minimalny stosunek kontrastu wg WCAG 2.1:

  • Normalny tekst: 4.5:1
  • Duży tekst (18px+): 3:1
  • Elementy UI (przyciski, ikony): 3:1

5. Nawigacja klawiaturą

Cała strona powinna być obsługiwalna samą klawiaturą. Sprawdź: czy możesz przejść przez wszystkie interaktywne elementy klawiszem Tab? Czy widać który element jest aktywny (focus)? 

<!-- Nigdy nie usuwaj outline bez alternatywy! -->
/* ❌ Źle */
* { outline: none; }

/* ✅ Dobrze — zastąp własnym stylem */
button:focus-visible { outline: 3px solid #0066cc; }

Narzędzia do sprawdzania dostępności

  • WAVE — rozszerzenie przeglądarki, wizualizuje błędy dostępności
  • axe DevTools — wtyczka do Chrome/Firefox
  • Lighthouse — wbudowany w Chrome DevTools (zakładka Accessibility)
  • NVDA / VoiceOver — czytniki ekranowe do testowania
💡 Dobra dostępność pomaga nie tylko osobom z niepełnosprawnościami. Strona dostępna jest też lepsza dla użytkowników mobilnych, osób starszych, osób w trudnych warunkach (słabe oświetlenie, głośne otoczenie) i robotów wyszukiwarek.
