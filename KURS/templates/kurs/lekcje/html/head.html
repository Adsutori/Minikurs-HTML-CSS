Sekcja <head>

Sekcja <head> to "zaplecze" dokumentu HTML — niewidoczna dla użytkownika, ale kluczowa dla działania strony. Zawiera metadane, linki do zasobów zewnętrznych i instrukcje dla przeglądarki.

Co może znaleźć się w <head>?

<head>
  <!-- Kodowanie znaków -->
  <meta charset="UTF-8">

  <!-- Responsywność -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- Tytuł strony -->
  <title>Strona główna | MojaStrona</title>

  <!-- Opis dla wyszukiwarek -->
  <meta name="description" content="Opis strony do 160 znaków">

  <!-- Arkusz stylów CSS -->
  <link rel="stylesheet" href="css/style.css">

  <!-- Favicon -->
  <link rel="icon" type="image/png" href="favicon.png">

  <!-- Czcionki Google Fonts -->
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link href="https://fonts.googleapis.com/css2?family=Inter&display=swap" rel="stylesheet">

  <!-- Style inline (niezalecane dla dużych projektów) -->
  <style>
    body { margin: 0; font-family: Inter, sans-serif; }
  </style>

  <!-- Skrypt JavaScript (defer = czeka na załadowanie HTML) -->
  <script src="js/main.js" defer></script>
</head>

Dołączanie CSS — trzy sposoby

<!-- 1. Zewnętrzny plik (najlepsza praktyka) -->
<link rel="stylesheet" href="style.css">

<!-- 2. Style wewnętrzne w <style> -->
<style>
  p { color: navy; }
</style>

<!-- 3. Inline na elemencie (ostateczność) -->
<p style="color: navy">...</p>

Dołączanie JavaScript — gdzie i jak?

<!-- W <head> z defer — plik ładuje się równolegle, wykonuje po HTML -->
<script src="main.js" defer></script>

<!-- W <head> z async — ładuje się równolegle, wykonuje natychmiast -->
<script src="analytics.js" async></script>

<!-- Na końcu <body> — stara metoda, dziś zastąpiona przez defer -->
<script src="main.js"></script>
MetodaKiedy używać
deferWiększość skryptów — czeka na DOM, zachowuje kolejność
asyncNiezależne skrypty (np. analytics) — nie czeka na DOM
Koniec <body>Stara praktyka — defer jest lepszy
💡 Kolejność elementów w <head> ma znaczenie. meta charset i meta viewport powinny być zawsze pierwsze — przed <title> i innymi tagami.
