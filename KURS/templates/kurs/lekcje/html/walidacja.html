Walidacja HTML

Walidacja HTML to sprawdzenie, czy kod jest zgodny ze standardami W3C. Poprawny kod działa przewidywalnie we wszystkich przeglądarkach, jest łatwiejszy w utrzymaniu i lepiej indeksowany przez wyszukiwarki.

Najczęstsze błędy HTML

Brakujący DOCTYPE

<!-- ❌ Źle — przeglądarka przechodzi w tryb quirks -->
<html>...</html>

<!-- ✅ Dobrze -->
<!DOCTYPE html>
<html lang="pl">...</html>

Niezamknięte znaczniki

<!-- ❌ Źle -->
<ul>
  <li>Punkt 1
  <li>Punkt 2
</ul>

<!-- ✅ Dobrze -->
<ul>
  <li>Punkt 1</li>
  <li>Punkt 2</li>
</ul>

Nieprawidłowe zagnieżdżenie

<!-- ❌ Źle -->
<p>Tekst <strong>ważny</p></strong>

<!-- ✅ Dobrze -->
<p>Tekst <strong>ważny</strong></p>

Brakujący atrybut alt

<!-- ❌ Źle -->
<img src="foto.jpg">

<!-- ✅ Dobrze -->
<img src="foto.jpg" alt="Opis zdjęcia">

Duplikaty ID

<!-- ❌ Źle — dwa elementy z tym samym id -->
<div id="sekcja">...</div>
<div id="sekcja">...</div>

<!-- ✅ Dobrze -->
<div id="sekcja-1">...</div>
<div id="sekcja-2">...</div>

Walidator W3C

Oficjalny walidator HTML dostępny jest pod adresem validator.w3.org. Możesz sprawdzić stronę na trzy sposoby:

  • By URL — podaj adres opublikowanej strony
  • By File Upload — wgraj plik HTML
  • By Direct Input — wklej kod HTML bezpośrednio

Walidacja w VS Code

Rozszerzenia, które pomagają pisać poprawny HTML na bieżąco:

  • HTMLHint — podkreśla błędy składniowe w czasie pisania
  • W3C Web Validator — waliduje plik przez API W3C
  • Prettier — automatycznie formatuje i poprawia wcięcia
💡 Przeglądarki są bardzo tolerancyjne — wyświetlą stronę nawet z błędami w HTML. Nie oznacza to jednak, że kod jest poprawny. Błędy mogą powodować różne zachowanie w różnych przeglądarkach, problemy z dostępnością i gorsze pozycje w SEO.
