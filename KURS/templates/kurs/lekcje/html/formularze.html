Formularze

Formularz to sposób na zbieranie danych od użytkownika — logowanie, rejestracja, wyszukiwarka, zamówienie. Wszystko opiera się na znaczniku <form>, który grupuje pola i definiuje sposób wysyłania danych.

Podstawowa struktura formularza

<form action="/wyslij" method="POST">
  <label for="imie">Imię:</label>
  <input type="text" id="imie" name="imie" required>

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>

  <button type="submit">Wyślij</button>
</form>

Atrybuty znacznika <form>

AtrybutOpis
actionAdres URL, na który wysyłane są dane formularza
method="GET"Dane dołączane do URL — widoczne, do wyszukiwania
method="POST"Dane w ciele żądania — niewidoczne, do logowania, rejestracji
enctypeKodowanie danych — wymagane multipart/form-data przy wysyłaniu plików
novalidateWyłącza wbudowaną walidację przeglądarki
autocompleteon / off — podpowiedzi dla całego formularza

GET vs POST

<!-- GET — dane widoczne w URL: /szukaj?q=html+kurs -->
<form action="/szukaj" method="GET">
  <input type="search" name="q" placeholder="Szukaj...">
  <button type="submit">Szukaj</button>
</form>

<!-- POST — dane ukryte w ciele żądania -->
<form action="/logowanie" method="POST">
  <input type="email" name="email">
  <input type="password" name="haslo">
  <button type="submit">Zaloguj się</button>
</form>

Label — powiązanie z polem

Każde pole formularza powinno mieć etykietę <label>. Powiązanie przez for i id sprawia, że kliknięcie etykiety aktywuje pole — ważne dla dostępności i wygody użytkownika. 

<!-- Metoda 1: przez for/id -->
<label for="haslo">Hasło:</label>
<input type="password" id="haslo" name="haslo">

<!-- Metoda 2: input wewnątrz label -->
<label>
  Hasło:
  <input type="password" name="haslo">
</label>

Textarea — pole wieloliniowe

<label for="wiadomosc">Wiadomość:</label>
<textarea id="wiadomosc" name="wiadomosc" rows="5" cols="40"
  placeholder="Wpisz swoją wiadomość..."></textarea>

Select — lista rozwijana

<label for="kraj">Kraj:</label>
<select id="kraj" name="kraj">
  <option value="">-- wybierz kraj --</option>
  <option value="pl">Polska</option>
  <option value="de">Niemcy</option>
  <option value="fr" selected>Francja</option>
</select>

Atrybut selected ustawia domyślnie wybraną opcję.

Grupowanie opcji — <optgroup>

<select name="jezyk">
  <optgroup label="Frontend">
    <option value="html">HTML</option>
    <option value="css">CSS</option>
    <option value="js">JavaScript</option>
  </optgroup>
  <optgroup label="Backend">
    <option value="python">Python</option>
    <option value="php">PHP</option>
  </optgroup>
</select>

Checkbox i Radio

<!-- Checkbox — wielokrotny wybór -->
<fieldset>
  <legend>Zainteresowania:</legend>
  <label><input type="checkbox" name="hobby" value="html"> HTML</label>
  <label><input type="checkbox" name="hobby" value="css"> CSS</label>
  <label><input type="checkbox" name="hobby" value="js"> JavaScript</label>
</fieldset>

<!-- Radio — jeden wybór z grupy -->
<fieldset>
  <legend>Poziom zaawansowania:</legend>
  <label><input type="radio" name="poziom" value="poczatkujacy"> Początkujący</label>
  <label><input type="radio" name="poziom" value="sredni"> Średniozaawansowany</label>
  <label><input type="radio" name="poziom" value="zaawansowany"> Zaawansowany</label>
</fieldset>

Przyciski radio z tą samą wartością name tworzą grupę — można wybrać tylko jeden z nich.

Fieldset i Legend — grupowanie pól

<form action="/rejestracja" method="POST">
  <fieldset>
    <legend>Dane osobowe</legend>
    <label for="imie">Imię:</label>
    <input type="text" id="imie" name="imie" required>
    <label for="nazwisko">Nazwisko:</label>
    <input type="text" id="nazwisko" name="nazwisko" required>
  </fieldset>

  <fieldset>
    <legend>Dane konta</legend>
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    <label for="haslo">Hasło:</label>
    <input type="password" id="haslo" name="haslo" minlength="8" required>
  </fieldset>

  <button type="submit">Zarejestruj się</button>
</form>

Przyciski — <button> vs <input>

<!-- Preferowany sposób — można wstawić HTML wewnątrz -->
<button type="submit">✅ Wyślij formularz</button>
<button type="reset">🗑️ Wyczyść</button>
<button type="button">Kliknij (bez akcji)</button>

<!-- Starszy sposób -->
<input type="submit" value="Wyślij">
💡 Walidacja HTML5: Atrybuty takie jak required, minlength, pattern czy type="email" dają podstawową walidację po stronie przeglądarki — bez JavaScript. To wygodne, ale nigdy nie zastępuje walidacji po stronie serwera.
