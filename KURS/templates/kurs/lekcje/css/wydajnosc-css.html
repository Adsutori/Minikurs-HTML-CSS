Wydajność CSS

Wolny CSS blokuje renderowanie strony. Zrozumienie jak przeglądarka przetwarza style pozwala pisać kod, który działa szybko nawet na słabszych urządzeniach i wolnych połączeniach.

Critical Rendering Path

Przeglądarka renderuje stronę w krokach:

1. Pobierz HTML → zbuduj DOM
2. Pobierz CSS  → zbuduj CSSOM
3. DOM + CSSOM  → Render Tree
4. Layout       → oblicz pozycje i rozmiary
5. Paint        → narysuj piksele
6. Composite    → złóż warstwy (GPU)

CSS jest render-blocking — przeglądarka nie wyświetli nic dopóki nie pobierze i nie przetworzy wszystkich arkuszy stylów.

Ładowanie CSS — optymalizacja

<!-- Krytyczny CSS — inline w <head> -->
<style>
  /* Tylko style potrzebne do pierwszego widoku (above the fold) */
  body { margin: 0; font-family: 'Inter', sans-serif; }
  .navbar { ... }
  .hero { ... }
</style>

<!-- Reszta CSS — ładowana asynchronicznie -->
<link rel="preload" href="main.css" as="style"
      onload="this.onload=null;this.rel='stylesheet'">
<noscript><link rel="stylesheet" href="main.css"></noscript>

<!-- Preconnect dla Google Fonts -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

Właściwości powodujące reflow (unikaj w animacjach)

/* Te właściwości wymuszają przeliczenie layoutu — WOLNE */
width, height, top, left, right, bottom
margin, padding, border-width
font-size, line-height
display, position, float
overflow

/* Te właściwości są akcelerowane GPU — SZYBKIE */
transform: translate(), rotate(), scale()
opacity
filter (częściowo)
will-change

will-change — optymalizacja warstw

/* Informuje przeglądarkę z wyprzedzeniem */
.animated-card {
  will-change: transform;
}

.fade-element {
  will-change: opacity;
}

/* Usuń po animacji — will-change zużywa pamięć */
element.addEventListener('animationend', () => {
  element.style.willChange = 'auto';
});

/* NIE rób tego — za dużo warstw */
* { will-change: transform; } /* ❌ */

contain — izolacja renderowania

/* Izoluje element od reszty dokumentu */
.widget {
  contain: layout;    /* zmiany wewnątrz nie wpływają na layout zewnętrzny */
  contain: paint;     /* nie maluj poza granicami */
  contain: style;     /* izolacja liczników i właściwości */
  contain: size;      /* rozmiar nie zależy od dzieci */
  contain: strict;    /* wszystko powyżej */
  contain: content;   /* layout + paint + style */
}

/* content-visibility — lazy rendering */
.below-fold-section {
  content-visibility: auto;
  contain-intrinsic-size: 0 500px; /* szacowana wysokość */
}
/* Przeglądarka pomija renderowanie niewidocznych sekcji */

Selektory — wydajność

/* Przeglądarka czyta selektory od prawej do lewej */

/* Wolne — zbyt ogólny klucz */
div > * > span { }        /* ❌ */
.nav ul li a span { }     /* ❌ za długi łańcuch */

/* Szybkie — konkretny klucz */
.nav-link { }             /* ✅ */
.nav-link span { }        /* ✅ akceptowalne */

/* Unikaj selektora uniwersalnego w złożonych regułach */
.container * { }          /* ❌ */
.container > .item { }    /* ✅ */

font-display — ładowanie czcionek

@font-face {
  font-family: 'Inter';
  src: url('inter.woff2') format('woff2');
  font-display: swap;     /* pokaż fallback, podmień gdy gotowe */
  /* swap | block | fallback | optional */
}

/* Google Fonts z display=swap */
@import url('https://fonts.googleapis.com/css2?family=Inter&display=swap');

Zmniejszanie CSS

/* Używaj skróconych zapisów */
margin: 16px 24px;           /* zamiast 4 właściwości */
border: 1px solid #333;      /* zamiast 3 właściwości */
background: #fff url() center / cover no-repeat;

/* Grupuj selektory */
h1, h2, h3 { line-height: 1.2; }

/* Zmienne CSS zamiast powtarzania wartości */
:root { --accent: #7c3aed; }
.btn  { background: var(--accent); }
.link { color: var(--accent); }
