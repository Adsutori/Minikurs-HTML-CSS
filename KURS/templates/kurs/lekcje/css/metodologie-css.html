Metodologie CSS

Gdy projekt rośnie, niezorganizowany CSS staje się koszmarem — konflikty nazw, rosnąca specyficzność, style trudne do usunięcia. Metodologie to sprawdzone konwencje organizacji kodu CSS w skalowalny sposób.

BEM — Block Element Modifier

Najpopularniejsza metodologia. Nazwy klas opisują strukturę: blok, element (część bloku) i modyfikator (wariant). 

/* Składnia: block__element--modifier */

/* Blok — samodzielny komponent */
.card { }
.navbar { }
.btn { }

/* Element — część bloku (__ podwójne podkreślenie) */
.card__image { }
.card__title { }
.card__footer { }
.navbar__logo { }
.navbar__link { }

/* Modyfikator — wariant (-- podwójny myślnik) */
.card--featured { }
.card--compact { }
.btn--primary { }
.btn--large { }
.navbar__link--active { }

/* HTML */
<div class="card card--featured">
  <img class="card__image" src="..." alt="...">
  <div class="card__body">
    <h2 class="card__title">Tytuł</h2>
    <p class="card__text">Opis</p>
  </div>
  <div class="card__footer">
    <a class="btn btn--primary btn--small">Czytaj więcej</a>
  </div>
</div>

SMACSS — Scalable and Modular Architecture

Dzieli CSS na 5 kategorii:

/* 1. Base — resetowanie, elementy HTML */
body { margin: 0; }
a { color: var(--accent); }

/* 2. Layout — główne sekcje strony (prefix l-) */
.l-header { position: fixed; top: 0; }
.l-sidebar { width: 240px; }
.l-main { margin-left: 240px; }

/* 3. Module — komponenty wielokrotnego użytku */
.card { ... }
.btn { ... }
.badge { ... }

/* 4. State — stany (prefix is-) */
.is-active { ... }
.is-hidden { display: none; }
.is-loading { opacity: 0.6; }

/* 5. Theme — motywy (prefix theme-) */
.theme-dark { ... }
.theme-light { ... }

ITCSS — Inverted Triangle CSS

Organizuje CSS od najbardziej ogólnego do najbardziej specyficznego:

/* 1. Settings — zmienne, konfiguracja */
/* 2. Tools — mixiny, funkcje (Sass) */
/* 3. Generic — reset, normalize */
/* 4. Elements — style elementów HTML */
/* 5. Objects — wzorce layoutu (prefix o-) */
/* 6. Components — komponenty UI (prefix c-) */
/* 7. Utilities — klasy pomocnicze (prefix u-) */

/* Przykład importu */
@import 'settings/colors';
@import 'settings/typography';
@import 'generic/reset';
@import 'elements/headings';
@import 'objects/container';
@import 'components/card';
@import 'components/button';
@import 'utilities/spacing';
@import 'utilities/display';

Utility-First — podejście Tailwind

/* Zamiast pisać CSS, używasz gotowych klas */

/* Tailwind CSS */
<div class="flex items-center gap-4 p-6 bg-white rounded-xl shadow-lg">
  <img class="w-16 h-16 rounded-full object-cover" src="..." alt="...">
  <div>
    <h2 class="text-xl font-bold text-gray-900">Jan Kowalski</h2>
    <p class="text-sm text-gray-500">Developer</p>
  </div>
</div>
/* Zalety: szybkość, spójność, brak konfliktów nazw */
/* Wady: długie klasy HTML, trudna czytelność */

CSS Modules

/* card.module.css */
.card { background: var(--bg-card); }
.title { font-size: 1.5rem; }

/* JavaScript (React) */
import styles from './card.module.css';

<div className={styles.card}>
  <h2 className={styles.title}>Tytuł</h2>
</div>

/* Generuje unikalne klasy: .card_abc123 */
/* Brak konfliktów nazw między komponentami */

Porównanie metodologii

Metodologia Najlepsza dla Krzywa uczenia
BEM Każdego projektu — prosta i skuteczna Niska
SMACSS Średnich projektów z zespołem Średnia
ITCSS Dużych projektów enterprise Wysoka
Utility-First Szybkiego prototypowania, React/Vue Średnia
CSS Modules Aplikacji komponentowych (React) Niska
💡 Praktyczna rada: Na start naucz się BEM — jest prosty, działa wszędzie i rozwiązuje 80% problemów z organizacją CSS. Reszta metodologii to rozszerzenia tej samej idei: nadaj klasom znaczące, przewidywalne nazwy i trzymaj się konwencji.
