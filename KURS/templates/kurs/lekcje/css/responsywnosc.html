Responsywność i Media Queries

Responsywny design to podejście, w którym strona dopasowuje swój wygląd do rozmiaru ekranu — od małego telefonu po duży monitor. CSS osiąga to głównie przez media queries i elastyczne jednostki.

Meta viewport — obowiązkowy

<!-- Bez tego media queries nie działają na mobile! -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Media Queries — składnia

/* Styl dla ekranów do 768px */
@media (max-width: 768px) {
  .container {
    padding: 0 16px;
  }
  .grid-3 {
    grid-template-columns: 1fr;
  }
}

/* Styl dla ekranów od 768px wzwyż */
@media (min-width: 768px) {
  .sidebar {
    display: block;
  }
}

/* Zakres */
@media (min-width: 768px) and (max-width: 1024px) {
  .grid-3 {
    grid-template-columns: repeat(2, 1fr);
  }
}

Mobile First vs Desktop First

/* Mobile First — zacznij od małego, dodawaj dla większych */
.grid {
  grid-template-columns: 1fr; /* telefon */
}
@media (min-width: 768px) {
  .grid { grid-template-columns: repeat(2, 1fr); } /* tablet */
}
@media (min-width: 1024px) {
  .grid { grid-template-columns: repeat(3, 1fr); } /* desktop */
}

/* Desktop First — zacznij od dużego, redukuj dla mniejszych */
.grid {
  grid-template-columns: repeat(3, 1fr); /* desktop */
}
@media (max-width: 1024px) {
  .grid { grid-template-columns: repeat(2, 1fr); } /* tablet */
}
@media (max-width: 768px) {
  .grid { grid-template-columns: 1fr; } /* telefon */
}

Mobile First jest zalecanym podejściem — większość ruchu pochodzi z telefonów, a przeglądarki mobilne renderują stronę szybciej gdy nie muszą nadpisywać dużej ilości styli.

Elastyczne jednostki

/* vw / vh — procent szerokości / wysokości viewport */
.hero {
  width: 100vw;
  min-height: 100vh;
}

.sidebar {
  width: 25vw;
  min-width: 200px;
  max-width: 320px;
}

/* clamp() — min | preferowany | max */
h1 {
  font-size: clamp(1.5rem, 4vw, 3rem);
}

.container {
  width: clamp(320px, 90%, 1200px);
  margin: 0 auto;
}

Responsywne obrazy

/* Podstawowe — nigdy nie wychodź poza rodzica */
img {
  max-width: 100%;
  height: auto;
  display: block;
}

/* srcset — różne rozdzielczości */
<img
  src="obrazek-800.jpg"
  srcset="obrazek-400.jpg 400w,
          obrazek-800.jpg 800w,
          obrazek-1200.jpg 1200w"
  sizes="(max-width: 768px) 100vw, 50vw"
  alt="Opis"
>

Media queries — inne typy

/* Orientacja */
@media (orientation: portrait)  { /* pionowo */ }
@media (orientation: landscape) { /* poziomo */ }

/* Preferencja ciemnego motywu */
@media (prefers-color-scheme: dark) {
  body { background: #0a0a0f; color: #f1f5f9; }
}

/* Preferencja zredukowanego ruchu */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

/* Wydruk */
@media print {
  .navbar, .sidebar { display: none; }
  body { color: black; background: white; }
}

Container Queries — nowoczesne podejście

/* Styl zależny od rozmiaru rodzica, nie viewport */
.card-wrapper {
  container-type: inline-size;
  container-name: card;
}

@container card (min-width: 400px) {
  .card {
    display: flex;
    flex-direction: row;
  }
}
💡 Złota zasada: Nie dodawaj breakpointów dla konkretnych urządzeń — dodawaj je tam, gdzie Twój layout się "łamie". Otwórz DevTools, zwęź okno i obserwuj kiedy design przestaje wyglądać dobrze.
